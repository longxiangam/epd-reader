# 第 7 篇：低功耗设计与电源管理

> 用 Rust 构建ESP32-C3 电子墨水屏阅读器 · 系列文章

## 引言

电池供电是嵌入式设备最基本的需求之一。本项目使用一块锂电池供电，ESP32-C3 在 WiFi 开启时功耗约 80-100mA，而深度睡眠时仅 5-10μA——差了四个数量级。

如何让用户在两次充电之间使用尽可能长的时间？答案有两个：一是利用电子墨水屏"断电不丢失"的特性，二是精心设计睡眠策略。本文将覆盖深度睡眠、RTC 内存、外设电源控制、电池管理等关键技术。

## 1. 深度睡眠

### 1.1 睡眠触发条件

系统在两种情况下进入深度睡眠：

```rust
// src/main.rs:252-254（主循环中）
loop {
    // ...
    to_sleep_tips(Duration::from_secs(0), Duration::from_secs(180), true).await;
    Timer::after(Duration::from_secs(5)).await;
}
```

主循环中，每 5 秒检查一次空闲时间。如果超过 180 秒（3 分钟）没有任何操作，就进入睡眠。

页面级别的睡眠时间可能不同：

```rust
// src/pages/main_page.rs:219
to_sleep_tips(Duration::from_secs(0), Duration::from_secs(30), true).await;
```

主菜单 30 秒无操作就睡眠，阅读页面则可配置（默认 120 秒）。

### 1.2 睡眠流程

`to_sleep_tips` 是睡眠的入口函数：

```rust
// src/sleep.rs:30-61
pub async fn to_sleep_tips(sleep_time: Duration, idle_time: Duration, show_sleep: bool) {
    if *LAST_ACTIVE_TIME.lock().await == Instant::MAX { return; }
    if Instant::now().duration_since(*LAST_ACTIVE_TIME.lock().await) > idle_time {
        // 1. 停止 WiFi
        force_stop_wifi().await;

        // 2. 配置唤醒源
        let wakeup_pins = unsafe { core::ptr::addr_of_mut!(WAKEUP_PINS).as_mut().unwrap() };
        let rtcio = RtcioWakeupSource::new(wakeup_pins.as_mut_slice());
        let wakeup_source = TimerWakeupSource::new(
            core::time::Duration::from_micros(sleep_time.as_micros())
        );

        // 3. 显示待机画面（电子墨水屏的亮点）
        if show_sleep {
            crate::display::show_sleep().await;
        }

        // 4. 保存当前 RTC 时间戳
        unsafe {
            *core::ptr::addr_of_mut!(WHEN_SLEEP_RTC_MS) = get_rtc_ms().await;
        }

        // 5. 关闭外设电源
        EINK_PWER_PIN.lock().await.take().unwrap().set_high();  // 关屏幕
        SD_PWER_PIN.lock().await.take().unwrap().set_high();    // 关 SD 卡

        // 6. 保存时间到 RTC 寄存器
        save_time_to_rtc().await;

        // 7. 进入深度睡眠
        RTC_MANGE.lock().await.as_mut().unwrap().sleep_deep(ws.as_slice());
    }
}
```

流程图：

```
空闲超时检测
    │
force_stop_wifi() ←── 停止 WiFi（最大省电项）
    │
配置唤醒源 ←── GPIO 按键 + 定时器
    │
show_sleep() ←── 在电子墨水屏上渲染待机画面
    │
保存 RTC 时间 ←── 记录进入睡眠的时间戳
    │
关闭外设电源 ←── 屏幕和 SD 卡分别关电
    │
save_time_to_rtc() ←── 将系统时间写入 RTC 寄存器
    │
sleep_deep() ←── CPU 停止，仅 RTC 保持供电
```

注意步骤 3 和 4 的顺序很重要：先在屏幕上显示待机画面，然后才关闭屏幕电源。因为电子墨水屏断电后画面不消失，所以这个画面会一直保持到下次唤醒——这就是"待机画面"的原理。

### 1.3 唤醒源

ESP32-C3 支持两种唤醒源：

**GPIO 唤醒**（按键唤醒）：

```rust
// src/main.rs:207-208
let rtc_io = static_cell::make_static!(rtc_pin);
add_rtcio(rtc_io, WakeupLevel::Low).await;  // 低电平唤醒（按键按下）
```

GPIO2 通过 ADC 分压同时连接了 Key2 和 Key3。按键按下时 GPIO2 被拉低，触发 RTC 唤醒。

**定时器唤醒**：

```rust
let wakeup_source = TimerWakeupSource::new(
    core::time::Duration::from_micros(sleep_time.as_micros())
);
```

定时器唤醒用于周期性更新（如天气数据）。如果 `sleep_time` 为 0，则不添加定时器唤醒源，只能通过按键唤醒。

### 1.4 唤醒后的恢复

唤醒后，程序从 `main()` 的开头重新执行（不是从睡眠点继续）。但关键状态通过 RTC 内存保持：

```rust
// src/main.rs:100-103
let reason = esp_hal::system::reset_reason().unwrap_or(...);
let wake_reason = esp_hal::system::wakeup_cause();
```

通过检查复位原因和唤醒原因，程序可以区分首次启动和深度睡眠唤醒，并恢复相应状态。

## 2. RTC 快速内存

RTC 快速内存是 ESP32-C3 上一块特殊的 SRAM，在深度睡眠期间保持供电。放在这里的变量在唤醒后仍然保留上次的值。

```rust
// src/display.rs:47-48
#[ram(unstable(rtc_fast))]
static mut RENDER_TIMES: u32 = 0;
```

本项目中有多个 RTC 变量：

| 变量 | 文件 | 用途 |
|------|------|------|
| `RENDER_TIMES` | display.rs | 渲染次数（决定是否全刷） |
| `LAST_BATTERY_PERCENT` | battery.rs | 上次电池电量 |
| `WHEN_SLEEP_RTC_MS` | sleep.rs | 入睡时的 RTC 时间戳 |
| `PAGE_INDEX` | main_page.rs | 当前选中的菜单项 |
| `PREV_BUFFER` | display.rs | 上一帧画面（2.7 寸屏） |

### RTC 内存的使用模式

```rust
// 写入
unsafe {
    *core::ptr::addr_of_mut!(WHEN_SLEEP_RTC_MS) = get_rtc_ms().await;
}

// 读取
pub async fn get_sleep_ms() -> u64 {
    get_rtc_ms().await - unsafe { *core::ptr::addr_of!(WHEN_SLEEP_RTC_MS) }
}
```

唤醒后，通过 `get_sleep_ms()` 计算睡眠了多长时间，加上睡眠前的系统时间，就能恢复当前的时钟——不需要每次唤醒都做 NTP 同步。

## 3. 外设电源控制

ESP32-C3 本身没有为每个外设提供独立的电源域，但本项目通过 GPIO 控制外部 MOSFET 来实现：

```rust
// src/main.rs:117-124
let mut eink_pwr_ctrl = Output::new(peripherals.GPIO21, Level::High, OutputConfig::default());
let mut sd_pwr_ctrl = Output::new(peripherals.GPIO1, Level::High, OutputConfig::default());

eink_pwr_ctrl.set_low();   // 低电平 = 开启电源（MOSFET 导通）
sd_pwr_ctrl.set_low();

// 存入全局，睡眠时可以关闭
EINK_PWER_PIN.lock().await.replace(eink_pwr_ctrl);
SD_PWER_PIN.lock().await.replace(sd_pwr_ctrl);
```

GPIO1 还需要 RTC 保持，确保深度睡眠期间引脚状态不变：

```rust
// src/main.rs:115
peripherals.GPIO1.rtcio_pad_hold(true);
```

`rtcio_pad_hold(true)` 告诉 RTC 在睡眠期间保持该引脚的当前电平。如果不设置，引脚在睡眠期间可能变为高阻态，导致外设意外上电。

## 4. WiFi 省电策略

WiFi 是最大的功耗来源。项目实现了自动关闭机制：

```
使用网络 → use_wifi()（加锁 + 刷新时间戳）
     │
完成 → finish_wifi()（解锁）
     │
do_stop 任务每 3 秒检查一次
     │
30 秒内没有新的网络请求？
     │
Yes → STOP_WIFI_SIGNAL → 断开 WiFi → WifiStopped
     │
下次需要网络时 → RECONNECT_WIFI_SIGNAL → 重新连接
```

这样，WiFi 只在实际需要时才开启。

## 5.1 ntp_worker 的定时策略

`ntp_worker` 是所有网络同步的调度中心。它的主循环间隔决定了网络请求的频率：

```rust
// src/worldtime.rs:337-384
loop {
    let mut sleep_sec = 3600;  // 默认 1 小时

    // NTP 同步：距上次同步超过 1 小时，或从未同步过
    if now - sync_time_second > 3600 || sync_time_second == 0 {
        match ntp_request(stack, clock).await {
            Ok(_) => { sleep_sec = 3600; }   // 成功：1 小时后再同步
            Err(_) => { sleep_sec = 1; }     // 失败：1 秒后重试
        }
    }

    // NTP 同步成功后，触发天气和节假日同步
    if sync_time_success() {
        Weather::sync_weather().await;
        HolidayInfo::sync_holiday().await;
    }

    Timer::after(Duration::from_secs(sleep_sec)).await;
}
```

天气同步的间隔由 `Weather::sync_weather()` 控制：

```rust
// src/weather.rs:128-138
pub async fn sync_weather() {
    // 首次启动，从 Flash 加载缓存的时间戳
    if WEATHER_SYNC_SECOND == 0 { Self::get_weather().await; }

    // 距上次天气同步超过 5 小时，才发起新请求
    if !sync_weather_success() || now - WEATHER_SYNC_SECOND > 3600 * 5 {
        let _ = Weather::request().await;
    }
}
```

实际的同步间隔：

| 同步项 | 间隔 | 说明 |
|--------|------|------|
| NTP 时间 | 1 小时 | `ntp_worker` 主循环 3600 秒 |
| 天气数据 | 5 小时 | `3600 * 5 = 18000` 秒 |
| 节假日 | 1 年 | 检查年份是否变化，变化时重新拉取 |
| NTP 失败重试 | 1 秒 | 快速重试，连续失败 10 次后降为 10 秒 |

WiFi 在每次同步完成后立即通过 `finish_wifi()` 释放，进入 30 秒倒计时自动关闭。以 5 小时一次天气更新计算，WiFi 每天只开启约 5 次，每次几秒。

## 5. 电池管理

### 5.1 ADC 电压检测

电池通过分压电路接到 GPIO4 的 ADC：

```rust
// src/main.rs:148
let bat_adc1_pin = adc1_config.enable_pin_with_cal::<_, AdcCalCurve<ADC1>>(
    peripherals.GPIO4, Attenuation::_11dB
);
```

ADC 使用曲线校准（`AdcCalCurve`）提高精度，11dB 衰减允许测量接近电源电压的信号。

### 5.2 电池百分比

```rust
// src/battery.rs
#[ram(unstable(rtc_fast))]
static mut LAST_BATTERY_PERCENT: u32 = 0;

pub struct Battery { adc_pin: ... }

impl Battery {
    pub fn get_percent(&mut self) -> u32 {
        // 读取 ADC → 电压 → 百分比
        let voltage = ...;  // 根据分压比计算
        let percent = (voltage - MIN_VOLTAGE) / (MAX_VOLTAGE - MIN_VOLTAGE) * 100;
        percent.clamp(0, 100)
    }
}
```

电池百分比存入 RTC 内存，在 UI 上显示。由于电子墨水屏刷新慢，电量信息不需要实时更新。

### 5.3 周期采样

```rust
#[embassy_executor::task]
pub async fn test_bat_adc() {
    loop {
        Timer::after(Duration::from_secs(60)).await;  // 每 60 秒采样一次
        // ... 读取 ADC，更新电量
    }
}
```

60 秒的采样周期平衡了实时性和功耗。

## 6. 功耗分析

各状态的估算功耗：

| 状态 | 功耗 | 持续时间 |
|------|------|---------|
| WiFi 开启 + 渲染 | ~100mA | 每次几秒 |
| 渲染（无 WiFi） | ~30mA | 每次几百毫秒 |
| 待机（CPU 运行，屏灭） | ~10mA | 短暂 |
| 深度睡眠 | ~10μA | 大部分时间 |

以 1000mAh 锂电池为例：
- 天气刷新每 5 小时一次（WiFi 开启约 5 秒）：100mA × 5s / 18000s ≈ 0.028mA 平均
- NTP 同步每小时一次（WiFi 开启约 3 秒）：100mA × 3s / 3600s ≈ 0.083mA 平均
- 用户交互（按键、翻页）按每天 30 分钟计：30mA × 0.5h / 24h ≈ 0.625mA 平均
- 深度睡眠 99%+ 的时间：10μA ≈ 可忽略
- **预估续航：数周到数月**（取决于使用频率）

如果只是偶尔阅读电子书（不频繁联网），续航可以达到数周甚至更长。

## 小结

| 技术点 | 要点 |
|--------|------|
| 深度睡眠 | `sleep_deep()` + RTC 保持供电 |
| 唤醒源 | GPIO RTC 唤醒 + 定时器唤醒 |
| RTC 内存 | `#[ram(unstable(rtc_fast))]`，深度睡眠保持 |
| 外设电源 | GPIO 控制 MOSFET，独立关电 |
| GPIO 保持 | `rtcio_pad_hold(true)`，睡眠期间保持引脚状态 |
| WiFi 省电 | 30 秒无请求自动关闭，需要时重连 |
| 电池 | ADC 采样 + 曲线校准，60 秒周期 |
| 电子墨水屏 | 断电不丢失，睡眠前渲染待机画面 |

低功耗设计的核心思路：**让 CPU 和外设尽可能多地在深度睡眠中度过，只在必要时短暂醒来工作**。电子墨水屏的"断电不丢失"特性与深度睡眠完美配合——睡眠前渲染的画面就是设备的"外观"。

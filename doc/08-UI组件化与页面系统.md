# 第 8 篇：UI 组件化与页面系统

> 用 Rust 构建ESP32-C3 电子墨水屏阅读器 · 系列文章

## 引言

在 296×128 或 400×300 像素的电子墨水屏上设计 UI，是一种独特的体验。没有色彩、没有动画、刷新慢——但你可以用像素精确控制每一个元素的位置。本项目的 UI 系统基于 `embedded-graphics` 框架，通过组件化和页面路由实现了清晰的应用结构。

## 1. embedded-graphics 与 Drawable trait

`embedded-graphics` 是 Rust 嵌入式图形的基石。它的核心是 `Drawable` trait：

```rust
// embedded-graphics 定义的 trait
pub trait Drawable {
    type Color: PixelColor;
    type Output;
    fn draw<D: DrawTarget<Color = Self::Color>>(
        &self,
        target: &mut D,
    ) -> Result<Self::Output, D::Error>;
}
```

任何实现了 `Drawable` 的类型都可以绘制到任意 `DrawTarget` 上——屏幕、帧缓冲区、甚至另一个组合组件。这种解耦让组件完全不知道自己最终绘制到哪里。

本项目所有的 UI 组件都实现了 `Drawable`：

```rust
// src/widgets/icon_grid_widget.rs:132-148
impl<C> Drawable for IconGridWidget<C>
where C: PixelColor + Clone
{
    type Color = C;
    type Output = ();

    fn draw<D>(&self, target: &mut D) -> Result<Self::Output, D::Error>
    where D: DrawTarget<Color = Self::Color>
    {
        for cell in self.cells.iter() {
            let _ = cell.draw(target);  // 每个格子也是 Drawable
        }
        Ok(())
    }
}
```

### 自动布局

网格根据列数和项目数量自动计算每个格子的大小：

```rust
// src/widgets/icon_grid_widget.rs:83-84
let cell_width = size.width / columns as u32;
let cell_height = size.height / ((items.len() + columns - 1) / columns) as u32;
```

在 2.9 寸屏（296×128）上，3 列布局下每个格子约 98×64 像素。

### 纯几何图标

由于电子墨水屏只有黑白两色，不能使用彩色图标。项目用 `embedded-graphics` 的基本图形（线条、矩形、圆形）绘制了所有图标：


这种纯几何的图标设计在电子墨水屏上效果很好——线条清晰、对比度高。

## 3. 温度趋势图（TempChart）

天气页面中的温度曲线是一个典型的数据可视化组件：

```rust
// src/widgets/temp_chart.rs:22-108
pub fn draw_temp_chart<C, D>(
    position: Point,
    chart_size: Size,
    points: &[TempPoint],    // 各天温度数据
    color: C,
    target: &mut D,
) -> Result<(), D::Error>
{
    // 1. 计算温度范围
    let temp_min = ... - 2;  // 留余量
    let temp_max = ... + 2;

    // 2. Y 坐标映射函数
    let temp_to_y = |temp: i32| -> i32 {
        chart_y + chart_h - ((temp - temp_min) * chart_h / temp_range)
    };

    // 3. 水平参考线（每 5°C 一条）
    let mut t = (temp_min / 5) * 5 + 5;
    while t < temp_max {
        Line::new(...).into_styled(stroke).draw(target)?;
        t += 5;
    }

    // 4. 高温折线 + 低温折线
    draw_polyline(&high_pts, thick_stroke, dot_fill, target)?;
    draw_polyline(&low_pts, thick_stroke, dot_fill, target)?;
}
```

折线绘制用点和线段的组合：

```rust
// src/widgets/temp_chart.rs:179-203
fn draw_polyline<C, D>(pts: &[Point], line_style, dot_style, target) {
    // 连线
    for i in 1..pts.len() {
        Line::new(pts[i - 1], pts[i]).into_styled(line_style).draw(target)?;
    }
    // 数据点（小圆）
    for &p in pts {
        Circle::new(Point::new(p.x - 2, p.y - 2), 4)
            .into_styled(dot_style).draw(target)?;
    }
}
```

## 5. Page Trait 与页面路由

### 5.1 Page Trait

所有页面都实现统一的 `Page` trait：

```rust
// src/pages/mod.rs:54-59
pub trait Page {
    fn new() -> Self;
    async fn render(&mut self);      // 渲染页面内容
    async fn run(&mut self, spawner: Spawner);  // 运行页面逻辑
    async fn bind_event(&mut self);  // 绑定事件监听
}
```

### 5.2 页面路由

`MainPage` 是页面路由器，它维护菜单列表和当前选中的页面：

```rust
// src/pages/main_page.rs:209-277（简化）
async fn run(&mut self, spawner: Spawner) {
    loop {
        if None == self.current_page {
            self.render().await;  // 渲染主菜单
            continue;
        }
        let menu_item = &self.menus[self.current_page.unwrap()];
        match menu_item.page_enum {
            EReadPage => {
                let mut read_page = ReadPage::new();
                read_page.bind_event().await;
                read_page.run(spawner).await;
                self.back().await;  // 返回后重新绑定主菜单事件
            }
            EWeatherPage => {
                let mut weather_page = WeatherPage::new();
                weather_page.bind_event().await;
                weather_page.run(spawner).await;
                self.back().await;
            }
            // ... 其他页面
        }
    }
}
```

页面生命周期：

```
MainPage.run()
  │
  ├── current_page == None → 渲染主菜单
  │
  └── current_page == Some(index)
        │
        创建子页面 → bind_event() → run() → 返回
        │
        back() → clear() + 重新绑定主菜单事件
```

### 5.3 事件绑定与清理

每次进入子页面时，先清除上一个页面的事件监听，再绑定新的事件：

```rust
// src/pages/main_page.rs:126-128
async fn bind_event(&mut self) {
    event::clear().await;  // 清除所有非常驻监听
    // 重新绑定主菜单的事件
    event::on(EventType::KeyShort(2), |info| { /* 下移 */ }).await;
    event::on(EventType::KeyShort(1), |info| { /* 上移 */ }).await;
    event::on(EventType::KeyShort(3), |info| { /* 进入 */ }).await;
}
```

`clear()` 只清除 `fixed: false` 的监听器，常驻监听器（如全局睡眠检测）不受影响。

## 6. 条件编译适配不同屏幕

不同尺寸的屏幕需要不同的 UI 参数。通过条件编译在编译期确定：

```rust
// src/epd2in9_txt.rs:15-34
#[cfg(feature = "epd2in9")]
const LINES_NUM: u32 = 7;      // 2.9 寸：7 行文字
#[cfg(feature = "epd2in9")]
pub const WIDTH: u32 = 296;
#[cfg(feature = "epd2in9")]
pub const HEIGHT: u32 = 128;

#[cfg(feature = "epd4in2")]
const LINES_NUM: u32 = 22;     // 4.2 寸：22 行文字
#[cfg(feature = "epd4in2")]
pub const WIDTH: u32 = 300;
#[cfg(feature = "epd4in2")]
pub const HEIGHT: u32 = 400;
```

这种编译期适配没有运行时开销——未选中的代码根本不会编译。

## 小结

| 技术点 | 要点 |
|--------|------|
| Drawable trait | 组件与绘制目标解耦，可复用 |
| 纯几何图标 | 无图片资源，适合电子墨水屏 |
| Page trait | 统一的页面接口：new/render/run/bind_event |
| 页面路由 | MainPage 作为路由器，match 分发到子页面 |
| 事件绑定 | 进入页面时 clear + bind，退出时 clear |
| 条件编译 | 编译期适配不同屏幕尺寸 |
| 裁剪区域 | embedded-graphics clipping 限制绘制范围 |

在极低分辨率下，UI 设计的核心是**信息密度和可读性**——每个像素都要有意义。组件化让不同页面可以复用相同的 UI 构建 blocks，而 Page trait 保证了统一的生命周期管理。
